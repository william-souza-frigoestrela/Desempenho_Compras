00 · Overview

Inventário de tabelas

O modelo conecta diretamente em dois bancos Oracle (DATAVALE e TOTVS_RM). A tabela-fato central é DCEVPB_SOLICITACAOTEMPOAPROVACAO, com 32 colunas cobrindo todo o ciclo: solicitação → cotação → aprovação → pedido. Duas dimensões de tempo (Calendario e Calendario2) servem a propósitos distintos — Calendario cobre 2023-2099 para análise de longo prazo; Calendario2 cobre apenas 2025-2026 e é usado pela maioria das medidas de período. Há 7 LocalDateTables auto-geradas e 1 DateTableTemplate, ambos criados automaticamente pela time intelligence nativa do Power BI.

Fonte de dados: Duas fontes Oracle distintas: datavale (servidor principal — fato + cadastro de empresas) e TOTVS_RM (RH — estrutura de setores e departamentos). A conexão é feita via Oracle.Database com queries SQL nativas. Sem paths locais — modelo pronto para ambientes com acesso às instâncias Oracle.

⚠ Dois calendários com papéis diferentes — risco de confusão

Calendario cobre 2023–2099 e tem relacionamento passivo com a fato. Calendario2 cobre apenas 2025–2026 e é o calendário ativo usado pela maioria das medidas. A maioria das medidas referencia Calendario2[Date] diretamente via MIN/MAX — não via relacionamento ativo. Isso pode causar resultados inesperados se o slicer de data estiver conectado ao Calendario em vez do Calendario2.

⚠ Medida "teste" em Calendario2 — resquício de desenvolvimento

A tabela Calendario2 contém uma medida chamada teste com lógica hardcoded para janeiro/2025. Deve ser removida antes de entregar o modelo em produção.

Tempo Médio (Aprovação x Pedido) AVERAGEX(FILTER(...), DATAHORAPEDIDO - SOLIAPROVACAO) → "DDd HHh MMm" +

displayFolder: _TelaEficiencia · Format: texto

O que faz: Calcula o tempo médio entre o envio da solicitação para aprovação (SOLIAPROVACAO) e a geração do pedido (DATAHORAPEDIDO), formatado como "DDd HHh MMm". Filtra apenas registros com ambas as datas preenchidas e com DTAAPROVACAOPEDIDO dentro do período de Calendario2. Retorna BLANK se não houver pedidos no período.

VAR TempoMedioDias =
    AVERAGEX(
        FILTER(DCEVPB_SOLICITACAOTEMPOAPROVACAO,
            NOT(ISBLANK([DATAHORAPEDIDO])) &&
            NOT(ISBLANK([SOLIAPROVACAO])) &&
            [DTAAPROVACAOPEDIDO] >= MIN(Calendario2[Date]) &&
            [DTAAPROVACAOPEDIDO] <= MAX(Calendario2[Date])
        ),
        [DATAHORAPEDIDO] - [SOLIAPROVACAO]
    )
VAR Dias = INT(TempoMedioDias)
VAR Horas = INT(MOD(TempoMedioDias * 24, 24))
VAR Minutos = INT(MOD(TempoMedioDias * 24 * 60, 60))
RETURN
IF([Pedidos Gerados no Período] < 1, BLANK(),
    FORMAT(Dias,"00") & "D " &
    FORMAT(Horas,"00") & "H " &
    FORMAT(Minutos,"00") & "M ")
